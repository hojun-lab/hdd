## Mission 1: Lifecycle Diagram

### 과제

커넥션 풀에서 커넥션 하나의 전체 생명주기를 다이어그램으로 그려라.

포함해야 할 **상태**:
- Created (생성됨)
- Idle (풀에서 대기 중)
- InUse (사용 중)
- Removed (폐기됨)

포함해야 할 **전이 조건** (화살표 라벨):
- 풀 초기화 시 생성
- getConnection() 호출
- release() 호출
- isValid() 실패
- maxLifetime 초과

### 형식

Mermaid `stateDiagram-v2`로 작성하라.

## Mission 2: Benchmark Report

### 과제

Week 1에서 측정한 v0(풀 없음)과 v1(ArrayList + synchronized)의 벤치마크 결과를 정리하라.

### 결과 표

아래 표를 채워라 (직접 측정한 수치 사용):

| 버전 | 조건 | 처리량 (req/sec) | 실패 | 핵심 교훈 |
|------|------|-----------------|------|----------|
| v0 | 50t/100i, 풀 없음 | | | |
| v0 | 250t/200i, 풀 없음 | | | |
| v1 | 50t/100i, pool=10 | | | |
| v1 | 50t/100i, pool=50 | | | |
| v1 | 50t/1000i, pool=10, sleep(100) | | | |

### 분석 (반드시 작성)

1. **v0 → v1에서 가장 큰 성능 차이를 만든 요인**은 무엇인가?
2. **v1에서 pool=10이 pool=50보다 빠른 이유**는 무엇인가?
3. **sleep(100)을 추가하면 실패율이 급증하는 이유**는 무엇인가? 이것이 프로덕션에서 의미하는 바는?
4. **v1의 한계**를 3가지 이상 나열하라.
